# 多模态论文思路总结

这个页面先不做“论文罗列”，而是做“方向地图”。读论文时优先看它解决了什么瓶颈、用了什么训练策略、补了什么能力缺口。

## 一眼看懂当前主线

近年的多模态论文，核心已经不只是“让模型看图说话”，而是继续往下面几条线推进：

- 从图文理解走向多模态推理
- 从单张图像走向视频、音频、3D、GUI 等复杂输入
- 从问答模型走向可执行的 Agent
- 从通用能力走向医学、科研、机器人等垂直场景
- 从“能回答”走向“回答可信、可对齐、可解释”

## 近年值得重点跟踪的思路

### 1. 空间推理增强

- 代表价值：让模型从“看见”走向“理解空间”
- 常见做法：构造大规模空间问答数据、加入深度或布局监督、引入量化空间描述
- 应用方向：机器人、导航、室内理解、自动驾驶

### 2. 工具使用与程序化推理

- 代表价值：把复杂任务拆成可执行步骤
- 常见做法：先生成程序或推理链，再蒸馏为端到端模型
- 应用方向：复杂视觉问答、图表理解、文档解析、跨工具任务

### 3. 多模态对齐与幻觉抑制

- 代表价值：降低语言先验压过视觉证据的问题
- 常见做法：多模态偏好优化、视觉条件约束、答案可验证训练
- 应用方向：图像问答、文档问答、医疗辅助分析

### 4. 视频理解与时序建模

- 代表价值：让模型理解“发生了什么”而不是“画面里有什么”
- 常见做法：关键帧压缩、时序 token 设计、长视频分段摘要、检索式记忆
- 应用方向：监控分析、教学视频理解、运动分析、人机交互

### 5. 多模态 Agent 与真实环境执行

- 代表价值：从 perception 模型变成 action 模型
- 常见做法：把屏幕视觉、文本指令、历史动作联合建模
- 应用方向：电脑助手、流程自动化、GUI 自动操作

### 6. 垂直领域多模态模型

- 代表价值：把通用基础能力迁移到高价值场景
- 常见做法：领域数据构造、专家标注、专门 benchmark
- 应用方向：医学影像、科研辅助、专业问答

## 读论文时建议固定回答这 6 个问题

1. 这篇论文解决的是“感知问题”还是“推理问题”？
2. 它的输入模态有哪些？
3. 它提升能力靠的是数据、结构、训练目标，还是工具链？
4. 它有没有解决多模态幻觉或视觉忽视问题？
5. 它的 benchmark 是离线问答，还是可执行任务？
6. 它适合成为“基础模型能力”，还是“垂直场景能力”？
